# Preferências do Usuário

## Idioma
- **Português (Brasil)** - todas as comunicações, nomes de commits, comentários e documentação

## Versionamento
- **Branch principal de trabalho:** `vibecode`
- Commits devem ser feitos no GitHub para cada alteração significativa
- Manter `checkpoint.md` atualizado com todos os commits

## Documentação
- Manter `memorias.md` como guia vivo do projeto
- Atualizar sempre que fizer alterações significativas
- `checkpoint.md` serve para saber exatamente onde parou na sessão anterior

## Estilo de Desenvolvimento
- **Cauteloso:** Não fazer nada que quebre o projeto
- **Consultivo:** Consultar antes de fazer mudanças drásticas
- **Consistente:** Manter coerência entre seções e componentes
- Seguir padrões existentes no código

## Projeto
- Sistema de chamados multi-empresa
- Foco em estabilidade e funcionalidade
- Tema escuro como padrão
- Sistema de roles (GOD > ADMIN > GESTOR > ATENDENTE)
